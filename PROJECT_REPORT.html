Preface

The current hiring market is highly competitive and heavily mediated by ATS screening, making resume optimization difficult for candidates and shortlisting inefficient for recruiters. Manual review at scale introduces bias and delay. CareerCraft AI was developed to solve this with AI-driven automation and decision support.

The platform replaces static resume builders with real-time resume scoring, ATS optimization, skill analysis, and AI-assisted content generation. For recruiters, it introduces an intelligent ranking engine paired with a visual Recruitment Analytics Dashboard to track pipeline quality and manage shortlisted talent. The system is built using a modern serverless full-stack architecture: Next.js (frontend), Firebase Auth/Firestore/Storage (backend), and Google Genkit (AI orchestration).

This report covers problem definition, system architecture, core feature implementation (including the new shortlisting and analytics engine), testing results, challenges, and scope of future upgrades, proving that AI can make hiring faster, scalable, and more objective.

Note: CareerCraft AI is not claimed to be a perfect solution—it is a functional MVP that demonstrates feasibility, automation impact, and clear expansion paths.

Chapter 2

Literature Survey

2.1 Overview of Project

CareerCraft AI is a serverless full-stack career platform built to optimize resumes for ATS, match candidates to jobs, and automate recruiter shortlisting. Users can build resumes with live preview, manage versions, and generate AI-optimized content.

Candidate AI Features

  • Resume Analyzer: Keyword/ATS feedback + fixes
  • ATS Optimizer: Score vs job description
  • Job Matcher: Role suggestions based on skills
  • Cover Letter Generator: Personalized drafts

Recruiter Features

  • Candidate Matcher: Bulk resume scoring & ranking
  • Shortlist Management: Persistent saving of top talent to Firestore
  • Recruitment Analytics: Visual score distribution and role breakdown charts

The system uses Next.js (frontend), Firebase Auth/Firestore/Storage (backend), Genkit with Google Gemini models (AI processing), and Razorpay webhooks (subscription verification & plan activation).

This is a working MVP focused on ranking accuracy, real-time analytics, and secure serverless architecture.

2.2 Existing System

Current career tools include job portals (LinkedIn, Indeed, Naukri), resume builders (Zety, Zety-like tools), and ATS platforms (Greenhouse, Level) operating separately.

Limitations for Candidates

  • Basic resume tools with no deep ATS feedback
  • ATS parsing errors break complex layouts
  • Rejection reasons are invisible
  • No personalized optimization guidance

Limitations for Recruiters

  • Excessive application volume
  • Crude keyword filtering
  • Manual shortlisting is time-consuming
  • Lack of visual data on candidate quality
  • Bias-prone review process

Key Problem: The system today is fragmented, non-transparent, and inefficient, with resume creation, job discovery, and screening working as disconnected workflows instead of one guided platform.

2.3 Need for New System

The current ecosystem lacks personalized ATS feedback, candidate ranking, and unified hiring workflow, resulting in wasted time for both candidates and recruiters.

CareerCraft AI Solutions

  • AI Resume Scoring & ATS Optimization: Clear, contextual improvement points
  • End-to-End Workflow: Resume build → analyze → job-match → apply inside one dashboard
  • Automated Candidate Ranking: Replace manual screening with intelligent AI scores
  • Recruiter Analytics: Real-time charts showing talent pool distribution
  • Admin Subscription Tracking: Manage 30-day cycles and manual role overrides

Priority Shift: From templates and reach → ATS success probability, job-fit intelligence, and recruitment analytics.

3.2 Objectives

This project was built with distinct, measurable deliverables for each user group.

For Job Seekers

  • Build resumes using a live-preview Resume Builder
  • Enable plan-based resume version control (limits depend on subscription)
  • Provide AI Resume Analysis with structured improvement points
  • Implement ATS Optimizer that scores resumes against job descriptions
  • Deliver Job Matching (skill-based role suggestions)
  • Support PDF export of final resume

For Recruiters

  • Rank multiple resumes using AI Candidate Matcher
  • Persistent Shortlisting: Save top candidates to a private recruiter dashboard
  • Recruitment Analytics: View visual charts of match score distributions and role breakdowns using recharts

Admin & System Level

  • Provide secure email auth using Firebase Authentication
  • Implement tiered subscriptions: Free, Essentials, Pro, Recruiter
  • Subscription Management: Monitor billing cycles, track expiry dates (30-day logic), and override user roles
  • Enable admin approval panel for pending upgrades
  • Maintain a responsive UI built with Next.js + Tailwind

3.3 Feasibility Study

1. Technical Feasibility

The platform is buildable with proven, well-documented services: Next.js for UI, Firebase for authentication/database/storage, Genkit with Google Gemini for AI processing, and Razorpay for payments using secure webhooks. The addition of recharts for analytics provides high-performance visualizations without heavy server overhead. Security rules ensure data privacy between users and recruiters.

2. Economic Feasibility

All development tools used are free or open-source. Firebase and Gemini provide pay-per-use scaling. The subscription model (₹199 - ₹999) creates a direct revenue path to sustain operational usage.

3. Operational Feasibility

The UI requires zero training for candidates. Recruiters can manage talent pools through a familiar tabbed interface. Admin governance is centralized in a subscription management panel, making the system low-maintenance.

4. Legal Feasibility

User data (resumes, proofs, profiles) is stored securely in Firebase. Razorpay terms and privacy policy compliance have been implemented. Secure webhooks prevent unauthorized plan activations.

Chapter 5

Result & Performance Analysis

5.1 Implementation Description

Implemented using a serverless architecture to ensure modularity and scalability.

1. Recruiter Module & Analytics Engine

  • Shortlisting: Candidates are persisted to a recruiter-owned subcollection via Firestore writes.
  • Visual Analytics: Uses recharts to perform real-time aggregation of the shortlist data.
  • Charts: BarChart for match score distribution and PieChart for job title breakdown.

2. Admin Subscription Management

  • Expiry Logic: Calculates 30-day intervals from planUpdatedAt using date-fns.
  • Manual Overrides: Provides a UI for admins to update user plans directly from the table.
  • Payment History: Tracks total revenue from successful plan activations.

3. System Traits

  • Real-time UI updates via Firestore onSnapshot listeners.
  • AI processing orchestrated by Genkit + Google Gemini.
  • Webhook-verified payment links for secure subscription scaling.
